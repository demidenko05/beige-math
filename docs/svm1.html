Understanding Support Vector Machines. Step 1. From points to vectors. Make this with your eyes.

Keywords: Support vector machines.

* In different sources vectors are described in slightly different notations, e.g. a vector is named with just bold lower case letter, see references.

Hyperplane. From points to vectors.

Hyperplane is a linear function described with this vector equation:

\begin{align} \vec{\mathbf{X}} \cdot \vec{\mathbf{W}} = 0 \end{align}

Where \(\vec{\mathbf{X}}\) is a data vector (e.g. points from origin (0,0)) in N-dimention. \(\vec{\mathbf{W}}\) is a coefficient vector. So, this is Dot-product of two vectors, i.e. for N-dimention this is:

\begin{align} \vec{\mathbf{X}} \cdot \vec{\mathbf{W}} = x1w1 + x2w2 + ... +xnwn \end{align}

For 2-D Hyperplane this is: x1w1 + x2w2 = 0, and this is equals x1w1/w1 + x2w2/w1 = 0, i.e. let w1=1 whatever, and we got simple equation: x1 + x2w2 = 0, i.e. x2 = -x1/w2

For standard geometry notation in dots this equation is a line function that goes through origin: y = ax, where a = -1/w2 in this case.

For 3D this is a plane that also goes trough origin (see MIT-MVC1)

To shift a function along the last axis, you should just add a shifting constant b (see PRECALC-SHIFT), and so does for a Hyperplane(line, plane, etc.). So, shifted Hyperplane equation is:

\begin{align} \vec{\mathbf{X}} \cdot \vec{\mathbf{W}} + b = 0 \end{align}

Dot product of two vectors is also:

\begin{align} \vec{\mathbf{X}} \cdot \vec{\mathbf{W}} = \| \vec{\mathbf{X}} \| \| \vec{\mathbf{W}} \| cos \varTheta \end{align}

This means that a non-shifted Hyperplane's vectors in dot product are perpendicular to each other, i.e. \(cos \varTheta = 0 ⇒ \varTheta = 90^o\). That is \(\vec{\mathbf{W}}\) is normal to \(\vec{\mathbf{X}}\).

Consider the simplest example - Hyperplane in 2D y=-x, that is x2=-x1/w2, that is {x1,x2}*{1,1}=0.
